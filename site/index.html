ClaudioOS

A bare-metal operating system for AI coding agents.
No Linux. No POSIX. No JavaScript.

Boots via UEFI on real hardware. Drives NVMe, AHCI, Intel NICs, USB 3.0, and NVIDIA GPUs from Ring 0. Authenticates to claude.ai via OAuth, runs multi-agent sessions with split-pane terminals on a 2560x1600 framebuffer. Ships a post-quantum SSH daemon, three language interpreters, and a text-mode web browser. Everything is #![no_std] Rust.

// Capabilities

Every driver, protocol, and runtime is written from scratch in #![no_std] Rust. No Linux kernel modules. No libc. No userspace.

Boot & Core

UEFI + SMP Multi-Core

UEFI boot with GDT/IDT, 48 MB heap, ACPI table parsing, and symmetric multiprocessing. AP cores started via SIPI. 2560x1600 GOP framebuffer with Terminus bitmap font.

Storage

NVMe + AHCI + SATA

Native NVMe driver with admin/IO queue pairs. AHCI/SATA for legacy disks. VFS layer with auto-detection across ext4, btrfs, NTFS, and FAT32 filesystems.

Networking

Intel NIC + TCP/IP + TLS 1.3

Drivers for e1000, I219-V, and I225-V Intel NICs plus VirtIO-net. smoltcp TCP/IP stack with DHCP and DNS. TLS 1.3 with AES-128-GCM from Ring 0.

Security

Post-Quantum SSH Daemon

ML-KEM-768 + X25519 hybrid key exchange. ML-DSA-65 host keys. ChaCha20-Poly1305 transport. A fully post-quantum SSH server running on bare metal.

AI

Claude Multi-Agent Dashboard

OAuth device flow to claude.ai from bare metal. Max subscription integration. SSE streaming of Messages API. Split-pane terminal with per-agent viewports and tool loops (20 rounds).

Shell

AI-Native Shell

28 builtin commands plus natural language passthrough to Claude. Type English, get results. Full pipeline support with the interpreters and editor below.

USB

xHCI USB 3.0

Full xHCI host controller driver. Device enumeration, endpoint configuration, interrupt and bulk transfers. USB keyboard and storage support.

GPU

NVIDIA GPU Compute

BAR mapping, command submission, and compute dispatch on NVIDIA GPUs. MMIO register access from bare metal for GPU-accelerated workloads.

Audio

Intel HDA Audio

High Definition Audio controller driver with codec discovery, widget parsing, and PCM stream output. Sound from bare metal.

Languages

Python + JavaScript + Rust

Three interpreters/compilers running on bare metal. Python-lite (vars, loops, functions). JS-lite (ES5 core). Rust compiler via Cranelift JIT codegen. All no_std.

Browser

Wraith Text Browser

HTML parser with CSS selectors and form detection. HTTP/HTTPS transport over smoltcp. Character-grid renderer. Cloudflare challenge solver built in.

Editor

Nano-Like Text Editor

Buffer-gap text editor with syntax awareness. Integrated as the edit_file tool for Claude agents. ~400 lines, 11 tests.

// Built with Claude

ClaudioOS started with a question: What if an AI coding agent didn't need an operating system underneath it? What if it was the operating system?

Every line of this project was written in collaboration with Anthropic's Claude. The kernel boots, initializes hardware, authenticates to claude.ai, and spawns agent sessions that can write code, compile it (via Cranelift), execute Python and JavaScript, edit files, browse the web, and SSH into remote machines -- all from bare metal with zero OS dependencies.

The development itself is recursive: Claude helps build the OS that runs Claude. The project has grown from a simple UEFI bootloader to a 33-crate workspace with hardware drivers for NVMe, AHCI, Intel NICs, USB 3.0, HDA audio, and NVIDIA GPUs. It reads ext4, btrfs, and NTFS filesystems. It runs a post-quantum SSH daemon. It has its own shell with natural language understanding.

19 crates have been extracted and published as standalone open-source libraries, each usable in any #![no_std] Rust project. The goal isn't just an OS -- it's proving that AI-native systems programming is possible, practical, and productive.

// Build & Run

# Prerequisites
rustup target add x86_64-unknown-none
cargo install bootimage

# Build the kernel (33 crates)
cargo build

# Create bootable disk images
cargo run --manifest-path tools/image-builder/Cargo.toml -- \
    target/x86_64-unknown-none/debug/claudio-os

# Run in QEMU with full networking + TLS
qemu-system-x86_64 \
    -bios /usr/share/OVMF/OVMF_CODE.fd \
    -drive format=raw,file=target/x86_64-unknown-none/debug/claudio-os-uefi.img \
    -device virtio-net-pci,netdev=net0 \
    -netdev user,id=net0,hostfwd=tcp::5555-:5555 \
    -serial stdio -m 512M -smp 4 -cpu Haswell

-cpu Haswell is required for AES-NI instructions used by TLS 1.3. For real hardware, the OS has been tested on i9-11900K, Supermicro SYS-4028GR-TRT, HP Victus laptop, and Arch Linux boxes.
